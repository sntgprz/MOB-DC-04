Notification Center

iOS has a single Notification Center that’s available to us throughout our program. The notification center functions through a three step process:
1. Add an observer for a certain event. The observer listens for the event and runs a method when that event is executed. Adding an observer requires 4 options:
  - The observer: usually self
  - The selector: a string which is the function name we run when the event has been met, followed by a collon
  - The name: a string of the key that we’ll be listening for in scenarios where we listen for custom observers. This is a predefined variable if it’s an out of the box iOS notification.
  - Object: allows us to pass objects in our notification. You should not pass data through this object. This will usually be nil, unless you want to pass a custom message.
// Custom observer:
NSNotificationCenter.defaultCenter().addObserver(self,
            selector: "doSomethingFunction:",
            name: "doSomething",
            object: nil)

// Predefined observer
NSNotificationCenter.defaultCenter().addObserver(self,
            selector: "doSomething:",
            name: UITextFieldTextDidChangeNotification,
            object: nil)
2. Trigger the event when necessary. For example, when the user goes to a certain screen or when the user taps a button. This takes two options:
  - The name of the notification, which is the same as the name defined in the observer.
  - The object we pass through the notification. The same rule applies here as above. Try not to pass objects through notifications.
// Post notification example
NSNotificationCenter.defaultCenter().postNotificationName("doSomething", object: nil)
3. Final step is to execute the function triggered by the event. This is the function defined in the selector part of the observer minus the colon. The only requirement here is that we must pass the notification paremeter in this function. 
func doSomethingFunction(notification: NSNotification) {
    print("triggered function from notification")
}
